Web Scraping

Scraping Glassdoor UK job listings using Selenium Webdriver

View this project on Github

This is the first part of the Data Scientist Jobs Project, where I scraped data scientist job listings from Glassdoor. Python has a few libraries for scraping the web: Requests, lxml, and BeautifulSoup, which can be used to scrape data from static websites (where the same content is displayed to all visitors in the same format). These would not work for Glassdoor, which is a dynamic website, presenting different information to different visitors. Instead, I used Selenium WebDriver (which is compatible with Python) as was used by data scientist Ken Jee in the project that inspired this one.

Selenium WebDriver is mostly used for automating website testing; it allows you to simulate a user interacting with the website (clicking buttons, waiting for pages to load, etc).

The basics steps in the script I wrote to scrape Glassdoor with Selenium WebDriver are:

  1. Create an instance of Selenium WebDriver

  2. Navigate to the Glassdoor webpage showing search results for “data scientist” jobs in the UK posted within the last 30 days

  3. Look for elements on the webpage (e.g., pop-ups, lists, buttons, objects that hold text information) via “locators” in Selenium

  4. Perform one or more user actions on the element (in this script: either click or save text as a variable)

  5. Scrape and record the data for as many jobs in the search results as specified by the user of the script

  6. Depending on whether the script is used in the command line or the scraping function within the script is imported and used elsewhere, the jobs data are returned as a pandas DataFrame or saved as a comma-separated values (CSV) file, respectively The Glassdoor website regularly changed during the development of this script, so I used the Python logging module to record when a website change meant that the code was no longer successfully scraping from it.

Although the real-time aspect of it makes webscraping with Selenium Webdriver a slow process it was an effective tool for interacting with - and taking information from - dynamic web elements that any real user could see. View this project on Github
